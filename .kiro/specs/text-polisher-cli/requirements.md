# Text Polisher CLI - 要件定義

## 概要

適当な日本語テキストを、指定した丁寧さレベルとフォーマットに応じて丁寧な文章に変換するCLIツール。
裏側のAIエンジンとして Claude Code CLI または Kiro CLI をサブプロセスとして呼び出す。

## アーキテクチャ

- **構成**: シングルパッケージ（TypeScript）
- **パッケージマネージャー**: pnpm
- **AIエンジン**: Claude Code CLI (`claude`) or Kiro CLI (`kiro`) をサブプロセスで実行
- **API呼び出し**: なし（Claude Code / Kiro が裏で処理）
- **AWS SDK**: 不要
- **認証**: 不要（Claude Code / Kiro 側で管理済み）

## ユーザーストーリー

### US-1: 基本的なテキスト変換
**As a** ユーザー
**I want to** 適当な日本語テキストを丁寧な文章に変換したい
**So that** ビジネスや日常で適切な表現を素早く得られる

#### 受け入れ基準
- [ ] コマンド引数でテキストを渡せる（`text-polisher "明日休むわ"`）
- [ ] パイプでテキストを渡せる（`echo "明日休むわ" | text-polisher`）
- [ ] 変換後のテキストのみが標準出力に出力される
- [ ] 入力テキストが空の場合、エラーメッセージを表示して終了する

### US-2: 丁寧さレベルの指定
**As a** ユーザー
**I want to** 丁寧さのレベルを指定したい
**So that** 場面に応じた適切な敬語レベルの文章を得られる

#### 受け入れ基準
- [ ] `--level casual` でカジュアル丁寧な表現に変換される
- [ ] `--level polite` で標準的な丁寧語に変換される（デフォルト）
- [ ] `--level formal` でビジネス敬語に変換される
- [ ] `--level honorific` で最上級の敬語に変換される
- [ ] `--level` 未指定時は `polite` がデフォルトとして適用される

### US-3: フォーマットの指定
**As a** ユーザー
**I want to** 出力フォーマットを指定したい
**So that** 用途に応じた適切な形式の文章を得られる

#### 受け入れ基準
- [ ] `--format email` でメール形式（挨拶・本文・締め）に変換される
- [ ] `--format slack` でSlack向け（簡潔・絵文字適度）に変換される
- [ ] `--format document` で文書形式（正式・構造的）に変換される
- [ ] `--format` 未指定時はプレーンテキスト（フォーマットなし）で出力される

### US-4: オプションの組み合わせ
**As a** ユーザー
**I want to** レベルとフォーマットを組み合わせて指定したい
**So that** より細かく出力を制御できる

#### 受け入れ基準
- [ ] `--level formal --format email` のように組み合わせ可能
- [ ] 組み合わせ時、レベルとフォーマットの両方が反映された出力になる

### US-5: カスタムフォーマット
**As a** ユーザー
**I want to** 自分専用のフォーマット（文頭・文末の定型文など）を定義したい
**So that** よく使うパターンを再利用できる

#### 受け入れ基準
- [ ] `~/.text-polisher/formats.yaml` にカスタムフォーマットを定義できる
- [ ] カスタムフォーマットは `prefix`（文頭定型文）と `suffix`（文末定型文）を持つ
- [ ] `--format <カスタム名>` でカスタムフォーマットを呼び出せる
- [ ] カスタムフォーマットがビルトインと名前衝突した場合、カスタムが優先される
- [ ] 設定ファイルが存在しない場合でも正常に動作する（ビルトインのみ利用可能）

#### 設定ファイル例
```yaml
# ~/.text-polisher/formats.yaml
formats:
  moriyama-email:
    prefix: "お疲れ様です。森山です。"
    suffix: "以上です。よろしくお願いします。"

  weekly-report:
    prefix: "今週の報告です。"
    suffix: "以上、ご確認よろしくお願いいたします。"
```

#### 将来拡張（今回は未実装）

- ビルトインへのコントリビュート（PR）
- プラグインとしてnpm配布

### US-6: AIエンジンの選択
**As a** ユーザー
**I want to** 裏側で使うAIエンジンを選択したい
**So that** 自分の環境に合ったツールを使える

#### 受け入れ基準
- [ ] `--engine claude` で Claude Code CLI をサブプロセスとして使用する
- [ ] `--engine kiro` で Kiro CLI をサブプロセスとして使用する
- [ ] デフォルトエンジンは設定ファイルまたは環境変数で指定可能
- [ ] 指定されたエンジンのCLIがインストールされていない場合、エラーメッセージを表示して終了する

### US-7: エラーハンドリング
**As a** ユーザー
**I want to** エラー時に分かりやすいメッセージを見たい
**So that** 問題を把握して対処できる

#### 受け入れ基準
- [ ] エンジン（claude/kiro）が見つからない場合、インストール手順を表示して終了する
- [ ] 不正なオプション値が指定された場合、使用可能な値を表示して終了する
- [ ] エンジンの実行がエラーで終了した場合、エラー内容を表示して終了する
- [ ] `--help` で使い方が表示される

## 非機能要件

- ローカル利用のみ（npm公開しない）
- pnpm でパッケージ管理
- `pnpm link --global` でローカルインストールして使用
- Claude Code CLI または Kiro CLI がインストール済みであること前提

## 使用例

```bash
# 基本
text-polisher "明日休むわ"
# → 明日、お休みをいただきます。

# レベル指定
text-polisher --level formal "あの件よろしく"
# → あの件につきまして、よろしくお願いいたします。

# フォーマット指定（ビルトイン）
text-polisher --format email "明日の会議、資料送って"
# → お疲れ様です。
#   明日の会議の資料をお送りいただけますでしょうか。
#   よろしくお願いいたします。

# カスタムフォーマット指定
text-polisher --format moriyama-email "明日休むわ"
# → お疲れ様です。森山です。
#   明日、お休みをいただきます。
#   以上です。よろしくお願いします。

# エンジン指定
text-polisher --engine claude --level formal "明日休むわ"
text-polisher --engine kiro --level formal "明日休むわ"

# 組み合わせ
text-polisher --level honorific --format email "明日休むわ"

# パイプ
echo "確認しといて" | text-polisher --level formal

# ヘルプ
text-polisher --help
```
